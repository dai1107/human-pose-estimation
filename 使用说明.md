# 使用说明

本文档说明当前项目的实际使用流程。当前程序的核心思路是：

```text
先用摄像头采集一次运动学会话
再根据需要选择深蹲分析、篮球投篮分析，或与个人参考动作对比
```

系统只输出视觉姿态估计和运动学代理指标，不输出医学诊断、真实受力、真实关节力矩、肌肉发力大小，也不判断动作是否绝对标准。

## 1. 安装环境

在 PowerShell 中进入项目目录：

```powershell
cd "C:\Users\dell\Desktop\pose estimation"
```

创建并激活虚拟环境：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

`requirements.txt` 已包含 `ultralytics`，用于 `--backend yolo-pose`。默认 `mediapipe` 流程不会主动加载 YOLO；只有显式使用 YOLO Pose 或 `--person-detector yolo` 时才需要该依赖。

如果 PowerShell 不允许激活虚拟环境，可先执行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

## 2. 下载模型

程序默认读取：

```text
models/pose_landmarker_full.task
```

如果模型文件不存在，在 PowerShell 中执行：

```powershell
Invoke-WebRequest `
  -Uri "https://storage.googleapis.com/mediapipe-models/pose_landmarker/pose_landmarker_full/float16/latest/pose_landmarker_full.task" `
  -OutFile "models\pose_landmarker_full.task"
```

如果需要显示五根手指的手指点，还需要手部模型：

```text
models/hand_landmarker.task
```

如果手部模型文件不存在，在 PowerShell 中执行：

```powershell
Invoke-WebRequest `
  -Uri "https://storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/latest/hand_landmarker.task" `
  -OutFile "models\hand_landmarker.task"
```

## 4. 实时姿态检测

当前推荐先使用 `main.py` 做通用实时检测、视频复跑和 backend 对比：

```powershell
python main.py
python main.py --backend mediapipe --smoothing one-euro
python main.py --backend yolo-pose --smoothing one-euro
```

`main.py` 默认是 `--backend auto`。未知动作使用 MediaPipe；HYROX 已验证动作会按策略自动选择 MediaPipe 或 YOLO Pose。YOLO Pose 默认 `--yolo-device auto`，当前有 CUDA 时会使用 GPU。

实时窗口打开后，普通主链路可按 `B` 在 `mediapipe` 和 `yolo-pose` 之间切换，摄像头不会重新打开。该功能只支持 `--fusion none --person-detector none`；切换时会重置平滑器，避免不同关键点 schema 的上一帧状态混在一起。需要保存公平对比指标时，仍建议固定一个 backend 分别复跑同一段视频。

`main.py` 默认会增强实时稳定性：短暂识别不到 pose 时最多保留上一帧有效骨架 `5` 帧；手腕/手指靠近躯干或身体关节并引发异常跳变时，会短暂保留上一帧身体点。左上角出现 `tracking: HOLD` 或 `occlusion_guard` 表示保护逻辑正在兜底。若需要关闭，可加 `--pose-hold-frames 0` 或 `--no-occlusion-guard`。

按动作类型自动选择：

```powershell
python main.py --backend auto --action-type rowing --smoothing one-euro
```

按输入视频文件名自动选择：

```powershell
python main.py --headless --backend auto --input-video "HYROX视频\划船机.mp4" --smoothing one-euro --save-metrics results/rowing_auto.csv
```

保留的 HYROX 默认策略：

| 动作 | 推荐 backend |
|---|---|
| 农夫行走 | `mediapipe` |
| 划船机 | `yolo-pose` |
| 投掷药球 | `mediapipe` |
| 拉雪橇 | `yolo-pose` |
| 推雪橇 | `mediapipe` |
| 波比跳远 | `mediapipe` |
| 滑雪机 | `yolo-pose` |
| 负重箭步蹲 | `mediapipe` |

当前正式姿态 backend 只有 `mediapipe` 和 `yolo-pose`。`mediapipe` 输出 33 点，适合需要身体细节和后续运动学分析的动作；`yolo-pose` 输出 COCO 17 点，适合部分复杂背景或快速动作，默认 `--yolo-device auto`，有 CUDA 时使用 GPU。两者点位不同，不直接逐点平均。

`yolo-roi-mediapipe` 只是实验性辅助链路，不是第三个正式姿态模型：YOLO 只负责人体 bbox，姿态估计仍由 MediaPipe 输出 33 点。当前 HYROX 对比没有推荐它作为默认方案；只有多人、复杂背景或目标锁定需求明显时，才单独测试。

启动普通实时姿态检测：

```powershell
python app.py
```

或：

```powershell
python -m src.realtime_pose
```

常用参数：

```powershell
python app.py --camera 0 --width 1280 --height 720
python app.py --camera-fps 60 --max-detect-fps 30
python app.py --no-mirror
python app.py --metrics-overlay
python app.py --detect-width 480 --smoothing 0.75
python app.py --landmark-profile full
```

程序默认仍使用原来的 `pose_landmarker_full.task` 模型，但使用 `no-face` 姿态显示和保存配置，跳过鼻子、眼睛、耳朵、嘴部等面部点，减少绘制和 `landmarks.csv` 写入开销。需要完整 33 点时，可启动时加 `--landmark-profile full`，或运行中按 `F` 在显示/隐藏面部点之间切换。

实时检测默认请求 `640x480` 摄像头画面、`60 FPS`、`MJPG` 采集模式，并把姿态检测输入宽度降到 `480`、提交上限设为 `30 FPS`。如果启动日志里 reported FPS 仍接近 `10`，说明摄像头或当前环境可能只能稳定输出 10 FPS，可以继续降低分辨率或换摄像头支持的 FPS。

运行中按 `H` 会启用或关闭独立的 MediaPipe Hand Landmarker，显示五根手指的完整手指点，不显示手腕点；启动时加 `--show-hands` 只表示默认显示。当前显示和保存的是除 `wrist` 外的 20 个手指点：拇指 `CMC/MCP/IP/TIP`，食指/中指/无名指/小指 `MCP/PIP/DIP/TIP`。手部检测默认输入宽度是 `416`，用于降低延迟；如果手部点不够准，可以加 `--hand-detect-width 640`。

## 5. 实时快捷键

通用快捷键：

```text
Q / ESC：退出
B：在 main.py 普通实时主链路中切换 mediapipe / yolo-pose backend
S：保存截图
R：开始或停止视频录制
M：切换镜像
1：完整 33 点骨架模式
2：关键关节高亮模式
3：显示或隐藏运动学信息面板
F：显示或隐藏面部点
H：显示或隐藏手指点面板
C：开始或停止运动学数据采集会话
```

深蹲模式快捷键：

```text
K：开始或重新进行站立校准
P：开始或暂停深蹲专项分析
4：显示或隐藏深蹲专项信息面板
```

篮球模式快捷键：

```text
5：显示或隐藏篮球时序面板
J：开始或停止投篮片段候选采集标记
L：手动记录当前帧为出手代理时刻
```

## 6. 采集一次动作会话

启动实时程序后：

1. 面对摄像头完成准备。
2. 按 `C` 开始采集。
3. 完成动作，例如一次深蹲、一组深蹲、一次投篮或其他动作。
4. 再按 `C` 停止采集并保存。

保存目录示例：

```text
outputs/sessions/2026-07-04_162530/
├── metadata.json
├── landmarks.csv
├── kinematics.csv
├── summary.json
├── sequence_summary.json
├── angle_curves.png
└── velocity_curves.png
```

这里保存的是通用运动学数据。它本身不强制判断动作类型，后续再选择分析方式。`landmarks.csv` 默认按 `no-face` profile 保存，不写入面部点；需要完整 33 点时，启动时加 `--landmark-profile full`。

## 7. 深蹲实时模式

启动深蹲实时模式：

```powershell
python app.py --analysis-mode squat --camera-view side
```

如果深蹲时也需要显示手指点，启动后按 `H` 即可。

建议流程：

1. 站到画面中，确保肩、髋、膝、踝可见。
2. 按 `K` 做 2 到 3 秒站立校准。
3. 按 `P` 开始专项分析。
4. 做深蹲动作。
5. 如需保存原始会话数据，按 `C` 开始和停止采集。

实时深蹲模式只显示当前状态、次数和代理指标。完整报告建议使用离线分析。

## 8. 离线深蹲分析

## 9. 篮球投篮实时模式

启动篮球投篮实时模式：

```powershell
python app.py `
  --analysis-mode basketball `
  --shot-type set_shot `
  --shooting-side right `
  --camera-view side
```

可选参数：

```text
--shot-type set_shot 或 jump_shot
--shooting-side right 或 left
--camera-view side、front_left、front_right 或 unknown
```

实时篮球模式第一版只显示当前阶段和实时代理指标。完整阶段报告、出手代理、峰值事件、DTW 对齐和一致性分析建议使用离线工具完成。

## 10. 离线篮球投篮分析

先查看可能的投篮候选片段：

```powershell
python -m src.tools.inspect_shot `
  --session outputs\sessions\<session_id> `
  --shooting-side right
```

自动候选只是建议，推荐手动指定投篮片段：

```powershell
python -m src.tools.analyze_shot_session `
  --session outputs\sessions\<session_id> `
  --start-ms 1200 `
  --end-ms 3100 `
  --shot-type set_shot `
  --shooting-side right `
  --camera-view side `
  --output-dir outputs\basketball\reports
```

如果你人工确认了出手代理时刻，可指定：

```powershell
python -m src.tools.analyze_shot_session `
  --session outputs\sessions\<session_id> `
  --start-ms 1200 `
  --end-ms 3100 `
  --release-ms 1915 `
  --shot-type set_shot `
  --shooting-side right `
  --camera-view side
```

篮球报告输出示例：

```text
outputs/basketball/reports/<report_id>/
├── metadata.json
├── shot_summary.json
├── shot_events.csv
├── shot_features.csv
```

## 11. 建立个人参考动作

参考动作需要你主动选择和命名。程序不会自动认定某段动作是最佳动作。

通用参考动作：

```powershell

  --session outputs\sessions\<session_id> `
  --start-ms 1000 `
  --end-ms 3000 `
  --name "我的动作参考 A" `
  --action-type generic_motion `
  --camera-view side `
  --movement-side right
```

```powershell
python -m src.tools.create_reference `

  --end-ms 3000 `
```

## 13. 多次投篮一致性分析

先分别生成多个篮球投篮报告，然后比较：

```powershell
python -m src.tools.compare_shots `
  --shots outputs\basketball\reports\<shot_a> outputs\basketball\reports\<shot_b> outputs\basketball\reports\<shot_c> `
  --shooting-side right `
  --shot-type set_shot
```

一致性只表示多次动作之间的运动学相似程度，不代表投篮命中率或技术水平。

## 14. 常见工作流

### A. 分析一次深蹲

```text
启动 app.py
按 C 采集一段深蹲
按 C 停止保存
运行 analyze_squat_session
查看 outputs/squat_reports/<report_id>/report.md
```

### B. 分析一次投篮

```text
启动 app.py
按 C 采集一次投篮
按 C 停止保存
运行 inspect_shot 查看候选片段
手动选择 start-ms 和 end-ms
运行 analyze_shot_session
查看 outputs/basketball/reports/<report_id>/report.md
```

### C. 与自己的参考动作比较

```text
先录一段你认可的动作
用 create_reference 或 create_shot_reference 建立参考
以后录新的动作
用 compare_session、analyze_squat_session --reference 或 analyze_shot_session --reference 比较
```

## 15. 重要限制

当前系统不能直接测量：

```text
地面反作用力
真实关节力矩
肌肉发力大小
肌电活动
医学风险或运动损伤
篮球真实离手时刻
投篮命中率
动作是否绝对标准
```

`release_proxy_time` 只是基于人体关键点的出手代理时刻。只有未来接入并验证篮球检测后，才可以进一步标记篮球相关的 release candidate。
