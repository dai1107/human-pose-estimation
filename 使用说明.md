# 使用说明

本文档说明当前项目的实际使用流程。当前程序的核心思路是：

```text
先用摄像头采集一次运动学会话
再根据需要选择深蹲分析、篮球投篮分析，或与个人参考动作对比
```

系统只输出视觉姿态估计和运动学代理指标，不输出医学诊断、真实受力、真实关节力矩、肌肉发力大小，也不判断动作是否绝对标准。

## 1. 安装环境

在 PowerShell 中进入项目目录：

```powershell
cd "C:\Users\dell\Desktop\pose estimation"
```

创建并激活虚拟环境：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

如果 PowerShell 不允许激活虚拟环境，可先执行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

## 2. 下载姿态模型

程序默认读取：

```text
models/pose_landmarker_full.task
```

如果模型文件不存在，在 PowerShell 中执行：

```powershell
Invoke-WebRequest `
  -Uri "https://storage.googleapis.com/mediapipe-models/pose_landmarker/pose_landmarker_full/float16/latest/pose_landmarker_full.task" `
  -OutFile "models\pose_landmarker_full.task"
```

## 3. 基础检查

运行导入测试：

```powershell
python -m src.import_test
```

运行全部单元测试：

```powershell
python -m pytest -q
```

## 4. 实时姿态检测

启动普通实时姿态检测：

```powershell
python app.py
```

或：

```powershell
python -m src.realtime_pose
```

常用参数：

```powershell
python app.py --camera 0 --width 1280 --height 720
python app.py --no-mirror
python app.py --metrics-overlay
python app.py --detect-width 640 --smoothing 0.75
```

## 5. 实时快捷键

通用快捷键：

```text
Q / ESC：退出
S：保存截图
R：开始或停止视频录制
M：切换镜像
1：完整骨架模式
2：关键关节高亮模式
3：显示或隐藏运动学信息面板
C：开始或停止运动学数据采集会话
```

深蹲模式快捷键：

```text
K：开始或重新进行站立校准
P：开始或暂停深蹲专项分析
4：显示或隐藏深蹲专项信息面板
```

篮球模式快捷键：

```text
5：显示或隐藏篮球时序面板
J：开始或停止投篮片段候选采集标记
L：手动记录当前帧为出手代理时刻
```

## 6. 采集一次动作会话

启动实时程序后：

1. 面对摄像头完成准备。
2. 按 `C` 开始采集。
3. 完成动作，例如一次深蹲、一组深蹲、一次投篮或其他动作。
4. 再按 `C` 停止采集并保存。

保存目录示例：

```text
outputs/sessions/2026-07-04_162530/
├── metadata.json
├── landmarks.csv
├── kinematics.csv
├── summary.json
├── sequence_summary.json
├── angle_curves.png
└── velocity_curves.png
```

这里保存的是通用运动学数据。它本身不强制判断动作类型，后续再选择分析方式。

## 7. 深蹲实时模式

启动深蹲实时模式：

```powershell
python app.py --analysis-mode squat --camera-view side
```

建议流程：

1. 站到画面中，确保肩、髋、膝、踝可见。
2. 按 `K` 做 2 到 3 秒站立校准。
3. 按 `P` 开始专项分析。
4. 做深蹲动作。
5. 如需保存原始会话数据，按 `C` 开始和停止采集。

实时深蹲模式只显示当前状态、次数和代理指标。完整报告建议使用离线分析。

## 8. 离线深蹲分析

对已保存 session 做深蹲分析：

```powershell
python -m src.tools.analyze_squat_session `
  --session outputs\sessions\<session_id> `
  --camera-view side `
  --output-dir outputs\squat_reports
```

可先检查站立校准：

```powershell
python -m src.tools.calibrate_squat `
  --session outputs\sessions\<session_id> `
  --camera-view side
```

深蹲报告输出示例：

```text
outputs/squat_reports/<report_id>/
├── metadata.json
├── squat_reps.csv
├── squat_frames.csv
├── squat_summary.json
├── rep_timeline.png
├── angle_curves_by_rep.png
├── symmetry_curves.png
├── annotated_keyframes/
└── report.md
```

## 9. 篮球投篮实时模式

启动篮球投篮实时模式：

```powershell
python app.py `
  --analysis-mode basketball `
  --shot-type set_shot `
  --shooting-side right `
  --camera-view side
```

可选参数：

```text
--shot-type set_shot 或 jump_shot
--shooting-side right 或 left
--camera-view side、front_left、front_right 或 unknown
```

实时篮球模式第一版只显示当前阶段和实时代理指标。完整阶段报告、出手代理、峰值事件、DTW 对齐和一致性分析建议使用离线工具完成。

## 10. 离线篮球投篮分析

先查看可能的投篮候选片段：

```powershell
python -m src.tools.inspect_shot `
  --session outputs\sessions\<session_id> `
  --shooting-side right
```

自动候选只是建议，推荐手动指定投篮片段：

```powershell
python -m src.tools.analyze_shot_session `
  --session outputs\sessions\<session_id> `
  --start-ms 1200 `
  --end-ms 3100 `
  --shot-type set_shot `
  --shooting-side right `
  --camera-view side `
  --output-dir outputs\basketball\reports
```

如果你人工确认了出手代理时刻，可指定：

```powershell
python -m src.tools.analyze_shot_session `
  --session outputs\sessions\<session_id> `
  --start-ms 1200 `
  --end-ms 3100 `
  --release-ms 1915 `
  --shot-type set_shot `
  --shooting-side right `
  --camera-view side
```

篮球报告输出示例：

```text
outputs/basketball/reports/<report_id>/
├── metadata.json
├── shot_summary.json
├── shot_events.csv
├── shot_features.csv
├── chain_sequence.json
├── phase_timeline.png
├── angle_curves.png
├── velocity_curves.png
├── event_sequence.png
├── arm_path.png
├── keyframes/
└── report.md
```

## 11. 建立个人参考动作

参考动作需要你主动选择和命名。程序不会自动认定某段动作是最佳动作。

通用参考动作：

```powershell
python -m src.tools.create_reference `
  --session outputs\sessions\<session_id> `
  --start-ms 1000 `
  --end-ms 3000 `
  --name "我的动作参考 A" `
  --action-type generic_motion `
  --camera-view side `
  --movement-side right
```

深蹲参考动作：

```powershell
python -m src.tools.create_reference `
  --session outputs\sessions\<session_id> `
  --start-ms 1000 `
  --end-ms 3000 `
  --name "我的深蹲参考" `
  --action-type squat `
  --camera-view side `
  --movement-side bilateral
```

篮球投篮参考动作：

```powershell
python -m src.tools.create_shot_reference `
  --session outputs\sessions\<session_id> `
  --start-ms 1200 `
  --end-ms 3100 `
  --name "我的定点投篮参考" `
  --shot-type set_shot `
  --shooting-side right `
  --camera-view side
```

列出已有参考动作：

```powershell
python -m src.tools.list_references --root outputs\references
```

篮球投篮参考默认保存在：

```text
outputs/basketball/references/
```

## 12. 与个人参考动作对比

通用动作对比：

```powershell
python -m src.tools.compare_session `
  --session outputs\sessions\<candidate_session_id> `
  --reference outputs\references\<reference_id> `
  --start-ms 1000 `
  --end-ms 3000 `
  --output-dir outputs\comparisons
```

深蹲与个人参考对比：

```powershell
python -m src.tools.analyze_squat_session `
  --session outputs\sessions\<session_id> `
  --camera-view side `
  --reference outputs\references\<squat_reference_id>
```

篮球投篮与个人参考对比：

```powershell
python -m src.tools.analyze_shot_session `
  --session outputs\sessions\<session_id> `
  --start-ms 1200 `
  --end-ms 3100 `
  --shot-type set_shot `
  --shooting-side right `
  --camera-view side `
  --reference outputs\basketball\references\<shot_reference_id>
```

## 13. 多次投篮一致性分析

先分别生成多个篮球投篮报告，然后比较：

```powershell
python -m src.tools.compare_shots `
  --shots outputs\basketball\reports\<shot_a> outputs\basketball\reports\<shot_b> outputs\basketball\reports\<shot_c> `
  --shooting-side right `
  --shot-type set_shot
```

一致性只表示多次动作之间的运动学相似程度，不代表投篮命中率或技术水平。

## 14. 常见工作流

### A. 分析一次深蹲

```text
启动 app.py
按 C 采集一段深蹲
按 C 停止保存
运行 analyze_squat_session
查看 outputs/squat_reports/<report_id>/report.md
```

### B. 分析一次投篮

```text
启动 app.py
按 C 采集一次投篮
按 C 停止保存
运行 inspect_shot 查看候选片段
手动选择 start-ms 和 end-ms
运行 analyze_shot_session
查看 outputs/basketball/reports/<report_id>/report.md
```

### C. 与自己的参考动作比较

```text
先录一段你认可的动作
用 create_reference 或 create_shot_reference 建立参考
以后录新的动作
用 compare_session、analyze_squat_session --reference 或 analyze_shot_session --reference 比较
```

## 15. 重要限制

当前系统不能直接测量：

```text
地面反作用力
真实关节力矩
肌肉发力大小
肌电活动
医学风险或运动损伤
篮球真实离手时刻
投篮命中率
动作是否绝对标准
```

`release_proxy_time` 只是基于人体关键点的出手代理时刻。只有未来接入并验证篮球检测后，才可以进一步标记篮球相关的 release candidate。
