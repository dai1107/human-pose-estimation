# 使用说明

本文档说明当前项目的实际使用流程。当前程序的核心思路是：

```text
先用摄像头采集一次运动学会话
再根据需要选择深蹲分析、篮球投篮分析，或与个人参考动作对比
```

系统只输出视觉姿态估计和运动学代理指标，不输出医学诊断、真实受力、真实关节力矩、肌肉发力大小，也不判断动作是否绝对标准。

## 1. 安装环境

在 PowerShell 中进入项目目录：

```powershell
cd "C:\Users\dell\Desktop\pose estimation"
```

创建并激活虚拟环境：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

`requirements.txt` 已包含 `ultralytics`，用于 `--backend yolo-pose`。默认 `mediapipe` 流程不会主动加载 YOLO；只有显式使用 YOLO Pose 或 `--person-detector yolo` 时才需要该依赖。

如果 PowerShell 不允许激活虚拟环境，可先执行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

程序默认读取：


## 4. 实时姿态检测

当前推荐先使用 `main.py` 做通用实时检测、视频复跑和 backend 对比：

```powershell
python main.py
python main.py --backend mediapipe --smoothing one-euro
python main.py --backend yolo-pose --smoothing one-euro
```

`main.py` 默认是 `--backend auto`。未知动作使用 MediaPipe；HYROX 已验证动作会按策略自动选择 MediaPipe 或 YOLO Pose。YOLO Pose 默认 `--yolo-device auto`，当前有 CUDA 时会使用 GPU。

实时窗口打开后，普通主链路可按 `B` 在 `mediapipe` 和 `yolo-pose` 之间切换，摄像头不会重新打开。该功能只支持 `--fusion none --person-detector none`；切换时会重置平滑器，避免不同关键点 schema 的上一帧状态混在一起。需要保存公平对比指标时，仍建议固定一个 backend 分别复跑同一段视频。

`main.py` 默认会增强实时稳定性：短暂识别不到 pose 时最多保留上一帧有效骨架 `5` 帧；手腕/手指靠近躯干或身体关节并引发异常跳变时，会短暂保留上一帧身体点。左上角出现 `tracking: HOLD` 或 `occlusion_guard` 表示保护逻辑正在兜底。若需要关闭，可加 `--pose-hold-frames 0` 或 `--no-occlusion-guard`。

按动作类型自动选择：

```powershell
python main.py --backend auto --action-type rowing --smoothing one-euro
```

按输入视频文件名自动选择：

```powershell
python main.py --headless --backend auto --input-video "HYROX视频\划船机.mp4" --smoothing one-euro --save-metrics results/rowing_auto.csv
```

保留的 HYROX 默认策略：

| 动作 | 推荐 backend |
|---|---|
| 农夫行走 | `mediapipe` |
| 划船机 | `yolo-pose` |
| 投掷药球 | `mediapipe` |
| 拉雪橇 | `yolo-pose` |
| 推雪橇 | `mediapipe` |
| 波比跳远 | `mediapipe` |
| 滑雪机 | `yolo-pose` |
| 负重箭步蹲 | `mediapipe` |

当前正式姿态 backend 只有 `mediapipe` 和 `yolo-pose`。`mediapipe` 输出 33 点，适合需要身体细节和后续运动学分析的动作；`yolo-pose` 输出 COCO 17 点，适合部分复杂背景或快速动作，默认 `--yolo-device auto`，有 CUDA 时使用 GPU。两者点位不同，不直接逐点平均。

`yolo-roi-mediapipe` 只是实验性辅助链路，不是第三个正式姿态模型：YOLO 只负责人体 bbox，姿态估计仍由 MediaPipe 输出 33 点。当前 HYROX 对比没有推荐它作为默认方案；只有多人、复杂背景或目标锁定需求明显时，才单独测试。

启动普通实时姿态检测：

```powershell
python main.py
```

或：

```powershell
python main.py --backend mediapipe --smoothing one-euro
python main.py --backend yolo-pose --smoothing one-euro
```

常用参数：

```powershell
python main.py --camera 0 --width 1280 --height 720
python main.py --camera-fps 60
python main.py --no-mirror
python main.py --metrics-overlay
python main.py --hyrox-debug
python main.py --hyrox-action lunge
python main.py --hyrox-action lunge --hyrox-sensitivity high
python main.py --show-hands --hand-detect-width 640
python main.py --landmark-profile no-face
python main.py --analysis-mode squat --camera-view side
python main.py --analysis-mode basketball --shot-type set_shot --shooting-side right --camera-view side
```

如果你当前主要在测 HYROX 弓步，最常用的是下面三条：

```powershell
python main.py --hyrox-action lunge
python main.py --hyrox-action lunge --hyrox-sensitivity high
python main.py --hyrox-action lunge --hyrox-debug
python main.py --hyrox-action lunge --hyrox-config configs/hyrox/lunge.yaml
```

区别如下：

- `python main.py --hyrox-action lunge`
  日常使用命令。开启弓步实时分析，显示 `action / phase / reps / tip`，灵敏度使用默认 `medium`。
- `python main.py --hyrox-action lunge --hyrox-sensitivity high`
  更敏感的测试命令。阶段更容易确认，适合动作幅度较小、先检查状态机能否跟上的场景，但误报概率也会更高。
- `python main.py --hyrox-action lunge --hyrox-debug`
  排错命令。在日常弓步分析基础上，额外显示 `visible / lknee / rknee / lhip / rhip / torso` 调试值，用来判断是特征本身有问题，还是阈值与状态机需要调整。
- `python main.py --hyrox-action lunge --hyrox-config configs/hyrox/lunge.yaml`
  配置文件命令。适合配合本地回放视频调阈值，不需要改代码。

建议直接这样记：

- 推荐日常使用：`python main.py --hyrox-action lunge`
- 推荐排错使用：`python main.py --hyrox-action lunge --hyrox-debug`

`main.py` 默认仍使用 `pose_landmarker_full.task`，但默认显示 profile 现在是 `full`。如果你希望减少面部点绘制或导出量，可启动时加 `--landmark-profile no-face`，或运行中按 `F` 在 `full` / `no-face` 之间切换；按 `2`、`6`、`7`、`8` 可快速切到 `shot`、`no-face`、`upper-body`、`lower-body`。

`main.py` 默认请求 `640x480` 摄像头画面、`60 FPS`、`MJPG` 采集模式。如果启动日志里 reported FPS 仍接近 `10`，说明摄像头或当前环境可能只能稳定输出 10 FPS，可以继续降低分辨率或换摄像头支持的 FPS。

如果你要录一小段原始视频做 HYROX 回放测试，现在有两种方式：

- 启动时直接加 `--record-raw outputs\recordings\lunge_raw.mp4`
- 程序运行后按 `T`，需要时再开始或停止原始视频录制

这样就不必全程录制，后续分析会更方便。

运行中按 `H` 会启用或关闭独立的 MediaPipe Hand Landmarker，显示五根手指的完整手指点，不显示手腕点；启动时加 `--show-hands` 只表示默认显示。当前显示和保存的是除 `wrist` 外的 20 个手指点：拇指 `CMC/MCP/IP/TIP`，食指/中指/无名指/小指 `MCP/PIP/DIP/TIP`。手部检测默认输入宽度是 `416`，默认提交上限是 `18 FPS`，最多检测 `2` 只手；如果手部点不够准，可以加 `--hand-detect-width 640`，如果想限制检测负载，可以调 `--max-hand-detect-fps` 和 `--max-hands`。

如果你需要 `--detect-width`、`--max-detect-fps` 这类更细的异步检测控制，仍可使用高级入口：

```powershell
python -m src.realtime_pose
python -m src.realtime_pose --camera-fps 60 --max-detect-fps 30
python -m src.realtime_pose --detect-width 480 --smoothing 0.75
```

## 5. 实时快捷键

通用快捷键：

```text
Q / ESC：退出
B：在 main.py 普通实时主链路中切换 mediapipe / yolo-pose backend
S：保存截图
R：开始或停止视频录制
T：开始或停止原始视频录制
M：切换镜像
1：完整 33 点骨架模式
2：关键关节高亮模式
3：显示或隐藏运动学信息面板
F：显示或隐藏面部点
6：切换到 no-face 模式
7：切换到 upper-body 模式
8：切换到 lower-body 模式
H：显示或隐藏手指点面板
C：开始或停止运动学数据采集会话
```

这三个最容易混淆：

- `R`：录制当前窗口里的标注视频，包含骨架和文字。
- `T`：录制原始摄像头视频，不带骨架和文字，更适合后续回放重跑。
- `C`：保存 session 数据，不是视频，而是 `landmarks.csv`、`kinematics.csv` 等结构化结果。

深蹲模式快捷键：

```text
K：开始或重新进行站立校准
P：开始或暂停深蹲专项分析
4：显示或隐藏深蹲专项信息面板
```

篮球模式快捷键：

```text
5：显示或隐藏篮球时序面板
J：开始或停止投篮片段候选采集标记
L：手动记录当前帧为出手代理时刻
```

## 5.1 实时面板和角度信息

普通 `main.py` 左上角状态栏默认显示的是 backend、FPS、延迟、成功率、录制状态等运行信息，不逐项显示关节角。

如果按 `3`，或启动时加 `--metrics-overlay`，右上角会出现通用运动学面板，显示：

```text
R ELBOW      右肘角
R KNEE       右膝角
R WRIST SPEED 右手腕速度代理
PELVIS SPEED 骨盆速度代理
ENERGY       全身运动能量代理
```

如果使用深蹲模式 `--analysis-mode squat`，右侧下方面板会额外显示：

```text
LEFT KNEE / RIGHT KNEE  左右膝角
TRUNK TILT              躯干前倾代理角
PELVIS DISPLACEMENT     骨盆位移代理
```

如果使用篮球模式 `--analysis-mode basketball`，右侧下方面板会额外显示：

```text
<shooting side> KNEE   投篮侧膝角
<shooting side> ELBOW  投篮侧肘角
PELVIS SPEED           骨盆速度代理
<shooting side> WRIST SPEED 投篮侧手腕速度代理
RELEASE PROXY          手动或自动记录的出手代理时刻
```

如果启动时加 `--hyrox-debug`，画面上方还会显示一组 HYROX 调试指标：

```text
visible
lknee / rknee
lhip / rhip
torso
```

如果启动时加 `--hyrox-action lunge`，画面还会显示 HYROX 弓步实时分析面板：

```text
action: Lunge
phase: stand / descent / bottom / ascent / unknown
reps
tip
```

这三种 HYROX 运行方式的侧重点不同：

- `--hyrox-action lunge`
  正常分析弓步动作，不额外显示调试特征。
- `--hyrox-action lunge --hyrox-sensitivity high`
  仍然是正常分析，但阶段判断更敏感。
- `--hyrox-action lunge --hyrox-debug`
  仍然是正常分析，同时把调试特征值直接画到屏幕上，便于排查。

当前同一帧最多显示 2 条提示；如果出现 `LOW_VISIBILITY`，只显示低可见度提示，并暂停其他判断。

`--hyrox-sensitivity` 支持：

- `low`：更保守，确认更慢，误报更少。
- `medium`：默认档，连续 3 帧确认阶段。
- `high`：更敏感，适合动作幅度较小的测试。

`--hyrox-config` 默认读取：

```text
configs/hyrox/lunge.yaml
```

当前至少支持这些阈值：

- `visibility_min`
- `stand_knee_angle_min`
- `bottom_knee_angle_max`
- `torso_lean_warn`
- `stable_frames`
- `rep_cooldown_ms`

如果配置文件不存在，程序会自动使用内置默认值，不会崩溃。当前 HYROX 动作面板会显示配置名，方便确认本次运行实际加载的是哪套阈值。

这里的角度都来自 2D/3D 关键点推导出的视觉运动学代理值，不是医学测角或动力学测量值。

## 5.2 HYROX 视频回放测试工具

第五轮新增了本地视频回放工具，专门用来验证弓步分析器在视频上的表现。它不是主流程，主流程仍然是摄像头实时分析。

基本用法：

```powershell
python tools/replay_hyrox_video.py --video "HYROX视频\弓步.mp4" --hyrox-action lunge
```

常用变体：

```powershell
python tools/replay_hyrox_video.py --video "HYROX视频\弓步.mp4" --hyrox-action lunge --speed 0.5
python tools/replay_hyrox_video.py --video "HYROX视频\弓步.mp4" --hyrox-action lunge --speed 2.0
python tools/replay_hyrox_video.py --video "HYROX视频\弓步.mp4" --hyrox-action lunge --save-debug-csv outputs\hyrox_replay_lunge.csv
python tools/replay_hyrox_video.py --video "HYROX视频\弓步.mp4" --hyrox-action lunge --hyrox-config configs/hyrox/lunge.yaml
```

参数说明：

- `--video`：本地视频路径，必填。
- `--hyrox-action lunge`：当前只支持弓步分析。
- `--hyrox-sensitivity low|medium|high`：和实时模式使用同一套灵敏度规则。
- `--hyrox-config`：和实时模式共用的弓步阈值配置文件。
- `--speed`：回放速度，`1.0` 是原速，`0.5` 是半速，`2.0` 是两倍速。
- `--save-debug-csv`：把每帧的特征值、阶段结果、次数和反馈保存成 CSV。

回放窗口会显示：

- 原视频画面。
- MediaPipe 骨架。
- 当前动作、阶段、次数和反馈。

按 `Q` 或 `ESC` 可以退出。

## 6. 采集一次动作会话

启动实时程序后：

1. 面对摄像头完成准备。
2. 按 `C` 开始采集。
3. 完成动作，例如一次深蹲、一组深蹲、一次投篮或其他动作。
4. 再按 `C` 停止采集并保存。

保存目录示例：

```text
outputs/sessions/2026-07-04_162530/
├── metadata.json
├── landmarks.csv
├── kinematics.csv
├── summary.json
├── sequence_summary.json
├── angle_curves.png
└── velocity_curves.png
```

这里保存的是通用运动学数据。它本身不强制判断动作类型，后续再选择分析方式。`landmarks.csv` 会按你开始会话时的当前显示 profile 保存；`main.py` 默认是 `full`，如果你在按 `C` 之前切到 `no-face`、`upper-body`、`lower-body` 或 `shot`，保存结果也会跟着切换。

## 7. 深蹲实时模式

启动深蹲实时模式：

```powershell
python main.py --analysis-mode squat --camera-view side
```

如果深蹲时也需要显示手指点，启动后按 `H` 即可。

建议流程：

1. 站到画面中，确保肩、髋、膝、踝可见。
2. 按 `K` 做 2 到 3 秒站立校准。
3. 按 `P` 开始专项分析。
4. 做深蹲动作。
5. 如需保存原始会话数据，按 `C` 开始和停止采集。

实时深蹲模式只显示当前状态、次数和代理指标。完整报告建议使用离线分析。

## 11. 建立个人参考动作

```powershell
  --action-type generic_motion `
  --camera-view side `
  --movement-side right
```

```powershell
python -m src.tools.create_reference `
```

## 13. 多次投篮一致性分析

先分别生成多个篮球投篮报告，然后比较：

```powershell
python -m src.tools.compare_shots `
  --shots outputs\basketball\reports\<shot_a> outputs\basketball\reports\<shot_b> outputs\basketball\reports\<shot_c> `
  --shooting-side right `
  --shot-type set_shot
```

一致性只表示多次动作之间的运动学相似程度，不代表投篮命中率或技术水平。

## 14. 常见工作流

```text

```

## 15. 重要限制

当前系统不能直接测量：

```text
地面反作用力
真实关节力矩
肌肉发力大小
肌电活动
医学风险或运动损伤
篮球真实离手时刻
投篮命中率
动作是否绝对标准
```

`release_proxy_time` 只是基于人体关键点的出手代理时刻。只有未来接入并验证篮球检测后，才可以进一步标记篮球相关的 release candidate。
