# 使用说明

本文档说明当前项目的实际使用流程。当前程序的核心思路是：

```text
先用摄像头采集一次运动学会话
再根据需要选择深蹲分析、篮球投篮分析，或与个人参考动作对比
```

系统只输出视觉姿态估计和运动学代理指标，不输出医学诊断、真实受力、真实关节力矩、肌肉发力大小，也不判断动作是否绝对标准。

## 功能与入口速查

| 想做什么 | 推荐命令或操作 |
|---|---|
| 检查环境、模型和输出目录 | `python -m src.doctor --strict` |
| 检查摄像头 0 | `python -m src.doctor --camera 0` |
| 启动普通实时骨架 | `python main.py --camera 0 --camera-view front` |
| 开启 HYROX 指导 | 增加 `--hyrox-action <动作名>`，或窗口中按 `A` |
| 切换正面/侧面 | 启动时 `--camera-view front|side`，运行中按 `V` |
| 切换动作 | 按 `A` 后按 `0–8`，或按 `N` |
| 显示运动学指标 | 增加 `--metrics-overlay`，或按 `3` |
| 录制原始视频 | `--record-raw <文件>`，或按 `T` |
| 保存运动学会话 | 按 `C` 开始，再按 `C` 停止 |
| 离线回放 HYROX | `python tools/replay_hyrox_video.py ...` |
| 实时深蹲 | `--analysis-mode squat --camera-view side|front` |
| 实时篮球投篮 | `--analysis-mode basketball --shooting-side right|left` |
| 个人参考动作比较 | `src.tools.create_reference` + `src.tools.compare_session` |
| 检查两台摄像头 | `tools/check_multicamera.py --camera 0:front --camera 1:side` |

项目使用的主要模型和方法：MediaPipe Pose Landmarker Full 33 点、YOLO11n Pose COCO 17 点、
可选 MediaPipe Hand Landmarker、One Euro/EMA 平滑、三点关节角、归一化位移、动作状态机、
连续帧防抖、计数冷却、正面/侧面规则过滤、站立校准、线性重采样和约束 DTW。技术原理、
模型适用范围和两个实时入口的区别见 `README.md` 的“核心模型与技术方法”。

## 最短成功流程

```powershell
python -m src.doctor --strict
python main.py --camera 0 --camera-view front
```

窗口打开后，让肩、髋、膝、踝都进入画面。然后：

1. 按 `A`，再按 `2`，切换到 Wall Ball 指导。
2. 确认动作面板显示 `view: front / front`，而不是 `unknown`。
3. 做一次完整动作，观察 `phase` 是否变化、`reps` 是否增加。
4. 按 `T` 可录原始视频；按 `C` 可开始/停止结构化会话。
5. 按 `Q` 正常退出，检查控制台是否输出 Metrics summary。

## 1. 安装环境

需要 Python 3.10 或更高版本。

在 PowerShell 中进入项目目录：

```powershell
cd "C:\Users\dell\Desktop\pose estimation"
```

创建并激活虚拟环境：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

`requirements.txt` 已包含 `ultralytics`，用于 `--backend yolo-pose`。默认 `mediapipe` 流程不会主动加载 YOLO；只有显式使用 YOLO Pose 或 `--person-detector yolo` 时才需要该依赖。

如果 PowerShell 不允许激活虚拟环境，可先执行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

### 3.1 如何判断是否正常运行

需要分三层判断，不能只看“窗口能打开”。

#### A. 环境正常

运行：

```powershell
python -m src.doctor --strict
```

正常表现：每个必需项为 `[PASS]`，最后一行是 `READY`。如果不使用手部或 YOLO，也可以不加
`--strict`，此时可选组件缺失显示 `SKIP`，不影响 MediaPipe 主流程。

#### B. 姿态检测正常

正常表现：

- 控制台打印 `Resolved backend: mediapipe` 或 `yolo-pose`。
- 窗口持续刷新，FPS 不是长期为 0，人体移动时骨架同步移动。
- 全身入镜后不持续显示 `person_lost`、`No pose` 或 `LOW_VISIBILITY`。
- `--metrics-overlay` 中关节角和速度会随动作变化，不是一直空值。
- 退出时打印 `Metrics summary`；成功率应与实际入镜时间相符。固定要求 100% 并不合理，因为转身、出画和遮挡都会降低成功率。

#### C. 动作分析正常且当前结果可用

- 动作面板显示正确的 `action`、配置名和 `view`。
- `phase` 随动作按合理顺序变化，不是在静止时快速跳动。
- 只有完成状态机要求的完整周期后 `reps` 才增加。
- 出现 `CAMERA_VIEW_REQUIRED` 时先按 `V` 或重新用 `--camera-view` 启动。
- 出现 `CAMERA_VIEW_LIMITED` 表示程序仍在运行，但当前机位只能给有限结论，应改用推荐视角。
- 持续 `LOW_VISIBILITY` 表示运行链路正常但输入质量不足，此时其他动作结论会被暂停，不能把它当作动作错误。

仓库回归基线可用于检查端到端链路：

| 样例 | 动作 | 帧数 | 当前计数/状态基线 |
|---|---|---:|---|
| `负重箭步蹲.mp4` | lunge | 133 | reps 1，最终 descent |
| `投掷药球.mp4` | wall_ball | 145 | reps 3，最终 stand |
| `农夫行走.mp4` | farmers_carry | 297 | reps 0，最终 carrying |
| `划船机.mp4` | rowing | 512 | reps 5，最终 catch |
| `滑雪机.mp4` | skierg | 103 | reps 1，最终 return |
| `波比跳远.mp4` | burpee_broad_jump | 501 | reps 2，最终 unknown |
| `推雪橇.mp4` | sled_push | 247 | step count 3，最终 drive |
| `拉雪橇.mp4` | sled_pull | 332 | reps 2，最终 recover |

模型库或阈值升级后结果可能小幅变化；明显偏离时应保存 debug CSV，检查可见度、视角、阶段和关键角度，不能直接调高灵敏度掩盖问题。

#### 常见异常速查

| 表现 | 常见原因 | 处理 |
|---|---|---|
| `MediaPipe model file not found` | 模型路径错误 | 检查 `models/pose_landmarker_full.task` 或传入 `--model` |
| 摄像头打不开 | 编号错误、被其他程序占用 | 运行 `python -m src.doctor --camera 0`，再尝试编号 1 |
| 有画面但没有骨架 | 人体未完整入镜、光照差、模型输出失败 | 退远、改善光照、先固定 `--backend mediapipe` 排查 |
| 骨架抖动 | 遮挡、动作过快、分辨率低 | 使用 One Euro、提高检测质量、减少遮挡；不要盲目增加 hold 帧数 |
| FPS 很低 | CPU/GPU能力、分辨率或手部检测负载 | 降低分辨率，关闭手部，比较 MediaPipe/YOLO 设备设置 |
| 阶段不变化 | 视角不匹配、关键点缺失、阈值过严 | 指定正确 `--camera-view`，启用 `--hyrox-debug` 并回放 CSV |
| 次数过多 | 阶段抖动、阈值过松、冷却过短 | 检查 `stable_frames` 与 cooldown，不要只调 sensitivity |
| 次数为 0 | 动作周期不完整、可见度不足、视角受限 | 检查阶段顺序、`LOW_VISIBILITY` 和 `CAMERA_VIEW_LIMITED` |
| 输出文件没有生成 | 未停止录制/会话、路径不可写 | 正常按键停止，运行 doctor 检查输出目录 |
| 双摄像头不同步 | USB 带宽、采集格式或设备帧率不同 | 用 `check_multicamera.py` 查看 skew；正式动作融合目前未启用 |

## 4. 实时姿态检测

当前推荐先使用 `main.py` 做通用实时检测、视频复跑和 backend 对比：

```powershell
python main.py
python main.py --backend mediapipe --smoothing one-euro
python main.py --backend yolo-pose --smoothing one-euro
```

`main.py` 默认是 `--backend auto`。未知动作使用 MediaPipe；HYROX 已验证动作会按策略自动选择 MediaPipe 或 YOLO Pose。YOLO Pose 默认 `--yolo-device auto`，当前有 CUDA 时会使用 GPU。

实时窗口打开后，普通主链路可按 `B` 在 `mediapipe` 和 `yolo-pose` 之间切换，摄像头不会重新打开。该功能只支持 `--fusion none --person-detector none`；切换时会重置平滑器，避免不同关键点 schema 的上一帧状态混在一起。需要保存公平对比指标时，仍建议固定一个 backend 分别复跑同一段视频。

`main.py` 默认会增强实时稳定性：短暂识别不到 pose 时最多保留上一帧有效骨架 `5` 帧；手腕/手指靠近躯干或身体关节并引发异常跳变时，会短暂保留上一帧身体点。左上角出现 `tracking: HOLD` 或 `occlusion_guard` 表示保护逻辑正在兜底。若需要关闭，可加 `--pose-hold-frames 0` 或 `--no-occlusion-guard`。

### 4.1 选择正面或侧面

请根据摄像头实际位置选择评价视角：

```powershell
python main.py --camera-view front --hyrox-action wall_ball
python main.py --camera-view side --hyrox-action lunge
```

- 正面模式主要使用左右对称、膝内扣、手脚同步等正面指标。
- 侧面模式主要使用动作深度、躯干倾角、髋折叠、伸展和前后位移等侧面指标。
- 运行中按 `V` 可切换正面/侧面；动作状态会重置，会话采集中不允许切换。
- `front_left` 和 `front_right` 使用正面评价档案。
- 强侧面动作在正面模式只给有限结论，并明确提示推荐侧面拍摄。

离线回放也必须传入实际视角，调试 CSV 会保存 `camera_view` 和 `view_profile`：

```powershell
python tools/replay_hyrox_video.py --video "HYROX视频\滑雪机.mp4" --hyrox-action skierg --camera-view front --save-debug-csv outputs/skierg_front.csv
```

如果以后连接正面和侧面两台摄像头，可先检查设备是否能同时工作：

```powershell
python tools/check_multicamera.py --camera 0:front:mirror --camera 1:side:no-mirror --frames 60
```

当前正式动作分析仍优先保证单摄像头。上述工具只检查多设备读帧和时间偏差，双视角融合路线见
`多摄像头设计与实施路线.md`。

### 4.2 摄像头开启后切换指导动作

摄像头运行期间按 `A` 打开动作菜单，然后按数字选择：

```text
0 关闭指导       1 负重箭步蹲     2 投掷药球
3 农夫行走       4 划船机         5 滑雪机
6 波比跳远       7 推雪橇         8 拉雪橇
```

按 `N` 可直接循环到下一个动作。每次切换都会清空上一动作阶段和次数，保留当前正面/侧面视角。
菜单打开时，数字键临时用于选择动作；关闭菜单后，数字键恢复原来的骨架和专项面板快捷键。
不采用终端输入动作名称，是因为终端输入会阻塞摄像头帧循环；数字菜单更适合实时运行，也能避免中英文输入法问题。

按动作类型自动选择：

```powershell
python main.py --backend auto --action-type rowing --smoothing one-euro
```

按输入视频文件名自动选择：

```powershell
python main.py --headless --backend auto --input-video "HYROX视频\划船机.mp4" --smoothing one-euro --save-metrics results/rowing_auto.csv
```

保留的 HYROX 默认策略：

| 动作 | 推荐 backend |
|---|---|
| 农夫行走 | `mediapipe` |
| 划船机 | `yolo-pose` |
| 投掷药球 | `mediapipe` |
| 拉雪橇 | `yolo-pose` |
| 推雪橇 | `mediapipe` |
| 波比跳远 | `mediapipe` |
| 滑雪机 | `yolo-pose` |
| 负重箭步蹲 | `mediapipe` |

当前正式姿态 backend 只有 `mediapipe` 和 `yolo-pose`。`mediapipe` 输出 33 点，适合需要身体细节和后续运动学分析的动作；`yolo-pose` 输出 COCO 17 点，适合部分复杂背景或快速动作，默认 `--yolo-device auto`，有 CUDA 时使用 GPU。两者点位不同，不直接逐点平均。

`yolo-roi-mediapipe` 只是实验性辅助链路，不是第三个正式姿态模型：YOLO 只负责人体 bbox，姿态估计仍由 MediaPipe 输出 33 点。当前 HYROX 对比没有推荐它作为默认方案；只有多人、复杂背景或目标锁定需求明显时，才单独测试。

启动普通实时姿态检测：

```powershell
python main.py
```

或：

```powershell
python main.py --backend mediapipe --smoothing one-euro
python main.py --backend yolo-pose --smoothing one-euro
```

常用参数：

```powershell
python main.py --camera 0 --width 1280 --height 720
python main.py --camera-fps 60
python main.py --no-mirror
python main.py --metrics-overlay
python main.py --normalized-pose-debug
python main.py --hyrox-debug
python main.py --hyrox-action lunge --camera-view side
python main.py --hyrox-action wall_ball --camera-view front
python main.py --hyrox-action lunge --camera-view side --hyrox-sensitivity high
python main.py --show-hands --hand-detect-width 640
python main.py --landmark-profile no-face
python main.py --analysis-mode squat --camera-view side
python main.py --analysis-mode basketball --shot-type set_shot --shooting-side right --camera-view side
```

需要检查 MediaPipe / YOLO Pose 是否正确转成统一输出时，可分别运行：

```powershell
python main.py --backend mediapipe --normalized-pose-debug
python main.py --backend yolo-pose --yolo-device auto --normalized-pose-debug
```

控制台会在第 1 帧和每 30 帧打印 `source / frame / timestamp / keypoints / confidence / latency / bbox`。这里的 `keypoints` 固定使用两个模型共有的 COCO 风格 17 点，`x/y` 和 `bbox` 都是像素坐标。两种后端单独运行，不会为了统一格式而同帧启动两套推理。

如果你当前主要在测 HYROX 弓步，最常用的是下面三条：

```powershell
python main.py --hyrox-action lunge --camera-view side
python main.py --hyrox-action lunge --camera-view side --hyrox-sensitivity high
python main.py --hyrox-action lunge --camera-view side --hyrox-debug
python main.py --hyrox-action lunge --camera-view side --hyrox-config configs/hyrox/lunge.yaml
```

区别如下：

- `python main.py --hyrox-action lunge --camera-view side`
  日常使用命令。开启弓步实时分析，显示 `action / phase / reps / tip`，灵敏度使用默认 `medium`。
- `python main.py --hyrox-action lunge --hyrox-sensitivity high`
  更敏感的测试命令。阶段更容易确认，适合动作幅度较小、先检查状态机能否跟上的场景，但误报概率也会更高。
- `python main.py --hyrox-action lunge --hyrox-debug`
  排错命令。在日常弓步分析基础上，额外显示 `visible / lknee / rknee / lhip / rhip / torso` 调试值，用来判断是特征本身有问题，还是阈值与状态机需要调整。
- `python main.py --hyrox-action lunge --hyrox-config configs/hyrox/lunge.yaml`
  配置文件命令。适合配合本地回放视频调阈值，不需要改代码。

建议直接这样记：

- 推荐日常使用：`python main.py --hyrox-action lunge --camera-view side`
- 推荐排错使用：`python main.py --hyrox-action lunge --camera-view side --hyrox-debug`

`main.py` 默认仍使用 `pose_landmarker_full.task`，但默认显示 profile 现在是 `full`。如果你希望减少面部点绘制或导出量，可启动时加 `--landmark-profile no-face`，或运行中按 `F` 在 `full` / `no-face` 之间切换；按 `2`、`6`、`7`、`8` 可快速切到 `shot`、`no-face`、`upper-body`、`lower-body`。

`main.py` 默认请求 `640x480` 摄像头画面、`60 FPS`、`MJPG` 采集模式。如果启动日志里 reported FPS 仍接近 `10`，说明摄像头或当前环境可能只能稳定输出 10 FPS，可以继续降低分辨率或换摄像头支持的 FPS。

如果你要录一小段原始视频做 HYROX 回放测试，现在有两种方式：

- 启动时直接加 `--record-raw outputs\recordings\lunge_raw.mp4`
- 程序运行后按 `T`，需要时再开始或停止原始视频录制

这样就不必全程录制，后续分析会更方便。

运行中按 `H` 会启用或关闭独立的 MediaPipe Hand Landmarker，显示五根手指的完整手指点，不显示手腕点；启动时加 `--show-hands` 只表示默认显示。当前显示和保存的是除 `wrist` 外的 20 个手指点：拇指 `CMC/MCP/IP/TIP`，食指/中指/无名指/小指 `MCP/PIP/DIP/TIP`。手部检测默认输入宽度是 `416`，默认提交上限是 `18 FPS`，最多检测 `2` 只手；如果手部点不够准，可以加 `--hand-detect-width 640`，如果想限制检测负载，可以调 `--max-hand-detect-fps` 和 `--max-hands`。

如果你需要 `--detect-width`、`--max-detect-fps` 这类更细的异步检测控制，仍可使用高级入口：

```powershell
python -m src.realtime_pose
python -m src.realtime_pose --camera-fps 60 --max-detect-fps 30
python -m src.realtime_pose --detect-width 480 --smoothing 0.75
```

## 5. 实时快捷键

通用快捷键：

```text
Q / ESC：退出
A：打开动作菜单，菜单中按 0–8 选择
N：快速循环到下一个 HYROX 动作
V：切换 front / side 评价视角
B：在 main.py 普通实时主链路中切换 mediapipe / yolo-pose backend
S：保存截图
R：开始或停止视频录制
T：开始或停止原始视频录制
M：切换镜像
1：完整 33 点骨架模式
2：关键关节高亮模式
3：显示或隐藏运动学信息面板
F：显示或隐藏面部点
6：切换到 no-face 模式
7：切换到 upper-body 模式
8：切换到 lower-body 模式
H：显示或隐藏手指点面板
C：开始或停止运动学数据采集会话
```

这三个最容易混淆：

- `R`：录制当前窗口里的标注视频，包含骨架和文字。
- `T`：录制原始摄像头视频，不带骨架和文字，更适合后续回放重跑。
- `C`：保存 session 数据，不是视频，而是 `landmarks.csv`、`kinematics.csv` 等结构化结果。

深蹲模式快捷键：

```text
K：开始或重新进行站立校准
P：开始或暂停深蹲专项分析
4：显示或隐藏深蹲专项信息面板
```

篮球模式快捷键：

```text
5：显示或隐藏篮球时序面板
J：开始或停止投篮片段候选采集标记
L：手动记录当前帧为出手代理时刻
```

### Lunge 动作模式

如果启动时加 `--hyrox-debug`，画面上方还会显示一组 HYROX 调试指标：

```text
visible
lknee / rknee
lhip / rhip
torso
```

如果启动时加 `--hyrox-action lunge`，画面还会显示 HYROX 弓步实时分析面板：

```text
action: Lunge
cfg: lunge_default
view: side / side
phase: stand / descent / bottom / ascent / unknown
reps
tip
```

HYROX 通用特征已包含左右肘/肩/髋/膝角、躯干倾角、肩髋高度差、身体中心与高度、手腕和脚踝位置/间距，以及全身、上下肢和左右侧可见度。位置与距离均按画面宽高归一化，特征缺失时不会中断实时流程。第一轮保持提取函数无状态，暂不输出速度字段；详细字段方向和验收命令见 `README.md` 的“HYROX 通用姿态特征”。

这三种 HYROX 运行方式的侧重点不同：

- `--hyrox-action lunge`
  正常分析弓步动作，不额外显示调试特征。
- `--hyrox-action lunge --hyrox-sensitivity high`
  仍然是正常分析，但阶段判断更敏感。
- `--hyrox-action lunge --hyrox-debug`
  仍然是正常分析，同时把调试特征值直接画到屏幕上，便于排查。

当前同一帧最多显示 2 条提示；如果出现 `LOW_VISIBILITY`，只显示低可见度提示，并暂停其他判断。

弓步默认 `medium` 评判标准：关键点可见度须达到 `0.45`；最小膝角达到 `150°` 视为站立，降至 `115°` 以下（已有站立基线时髋中心还须下降至少 `0.035`）视为最低位，稳定完成 `bottom -> stand` 才计 1 次。最低位膝角大于 `100°` 会提示深度不足；躯干倾斜超过 `20°` 会提示前倾过多；完成时最小膝角小于 `165°` 或最小髋角小于 `160°` 会提示站起伸展不足。提示不一定撤销计数；完整规则表见 `README.md` 的“Lunge 当前评判标准”。

建立站立基线后，`bottom` 会同时检查膝角和髋部相对下降量，避免仅弯膝但身体未下降时误判。中文提示使用系统中文字体绘制；字体不可用时仍会回退到 OpenCV 文本绘制，不影响实时主流程。

`--hyrox-sensitivity` 支持：

- `low`：更保守，确认更慢，误报更少。
- `medium`：默认档，连续 3 帧确认阶段。
- `high`：更敏感，适合动作幅度较小的测试。

`--hyrox-config` 留空时按动作自动读取：

```text
lunge -> configs/hyrox/lunge.yaml
wall_ball -> configs/hyrox/wall_ball.yaml
farmers_carry -> configs/hyrox/farmers_carry.yaml
rowing -> configs/hyrox/rowing.yaml
skierg -> configs/hyrox/skierg.yaml
burpee_broad_jump -> configs/hyrox/burpee_broad_jump.yaml
sled_push -> configs/hyrox/sled_push.yaml
sled_pull -> configs/hyrox/sled_pull.yaml
```

以下是 Lunge 常用阈值示例；其他动作字段见 `configs/hyrox/README.md` 和各自动作 YAML：

- `visibility_min`
- `stand_knee_angle_min`
- `bottom_knee_angle_max`
- `torso_lean_warn`
- `hip_motion_tolerance`
- `hip_drop_min`
- `stable_frames`
- `rep_cooldown_ms`

如果显式传入的 `--hyrox-config` 路径不存在，程序会明确报错并退出；未显式传入时，如果默认 YAML 缺失或个别字段缺少，分析器使用内置安全默认值。当前 HYROX 动作面板会显示配置名，方便确认本次运行实际加载的是哪套阈值。

### Farmers Carry 动作模式

推荐正面拍摄，保证肩、髋、膝、脚踝和双腕全部入镜：

```powershell
python main.py --hyrox-action farmers_carry --camera-view front
python main.py --hyrox-action farmers_carry --camera-view front --hyrox-debug
python tools/replay_hyrox_video.py --video "HYROX视频\农夫行走.mp4" --hyrox-action farmers_carry --camera-view front --headless
```

阶段为 `ready / carrying / rest / unknown`。该动作不按次数计数，因此面板 `reps` 保持 0。
程序用身体中心、踝间距和膝角的短时变化近似判断行走，用双腕相对髋部位置判断手臂是否自然下垂。

默认检查：可见度 `0.55`、肩髋左右高度差 `0.08`、躯干倾斜 `25°`、连续 3 帧确认，
静止 `1200 ms` 后进入 rest。提示包括左右倾斜、肩高不平、手臂未下垂、躯干倾斜和搬运晃动。
程序不检测负重器械，不判断重量、200 米距离或比赛完成。

### Wall Ball 人体动作模式

```powershell
python main.py --hyrox-action wall_ball --camera-view front
python main.py --hyrox-action wall_ball --hyrox-debug
python main.py --hyrox-action wall_ball --hyrox-config configs/hyrox/wall_ball.yaml
```

画面阶段依次使用 `stand / squat_down / bottom / throw_extension / reset`。程序只在“已站立 → 到达满足深度的 bottom → 回到站立或投掷伸展”后计数。

Wall Ball 默认 `medium` 评判标准：关键点可见度须达到 `0.45`；站立位要求最小膝角至少 `150°` 且较好一侧髋角至少 `145°`；最低位候选要求最小膝角不高于 `110°`，同时 `hip_knee_depth >= -0.05` 才满足有效深度。投掷伸展位还要求较好一侧肘角至少 `125°`、手腕高于肩至少 `0.03`。必须按顺序稳定完成站立、有效最低位、返回站立或投掷伸展才计 1 次。默认阶段连续 `3` 帧确认，计数冷却 `400 ms`；完整提示阈值见 `README.md` 的“Wall Ball 当前评判标准”。

当前提示：

- `SQUAT_NOT_DEEP`：髋部视觉高度未达到配置的深蹲要求。
- `KNEES_CAVE_IN`：仅用正面 2D 膝/踝宽度估计，属于低置信度 `warn`。
- `NOT_FULL_EXTENSION`：站起阶段髋膝伸展不足。
- `LOW_VISIBILITY`：请保证全身入镜，出现时暂停其他判断。

这一版不检测球，不判断球是否击中目标，也不判断目标高度或 100 次完成。`--hyrox-config` 留空时会自动读取 `configs/hyrox/wall_ball.yaml`；显式传入路径可覆盖。

仓库自带 `投掷药球.mp4` 使用默认 `medium + one-euro` 回放时，当前验收基线为处理 `145` 帧、最终 `stand`、`reps: 3`：

```powershell
python tools/replay_hyrox_video.py --video "HYROX视频\投掷药球.mp4" --hyrox-action wall_ball --camera-view front --headless
```

这里的角度都来自 2D/3D 关键点推导出的视觉运动学代理值，不是医学测角或动力学测量值。

### Rowing 动作模式

## 6. 采集一次动作会话

启动实时程序后：

1. 面对摄像头完成准备。
2. 按 `C` 开始采集。
3. 完成动作，例如一次深蹲、一组深蹲、一次投篮或其他动作。
4. 再按 `C` 停止采集并保存。

保存目录示例：

```text
outputs/sessions/2026-07-04_162530/
├── metadata.json
├── landmarks.csv
├── kinematics.csv
├── summary.json
├── sequence_summary.json
├── angle_curves.png
└── velocity_curves.png
```

这里保存的是通用运动学数据。它本身不强制判断动作类型，后续再选择分析方式。`landmarks.csv` 会按你开始会话时的当前显示 profile 保存；`main.py` 默认是 `full`，如果你在按 `C` 之前切到 `no-face`、`upper-body`、`lower-body` 或 `shot`，保存结果也会跟着切换。

## 7. 深蹲实时模式

启动深蹲实时模式：

```powershell
python main.py --analysis-mode squat --camera-view side
```

如果深蹲时也需要显示手指点，启动后按 `H` 即可。

建议流程：

1. 站到画面中，确保肩、髋、膝、踝可见。
2. 按 `K` 做 2 到 3 秒站立校准。
3. 按 `P` 开始专项分析。
4. 做深蹲动作。
5. 如需保存原始会话数据，按 `C` 开始和停止采集。

实时深蹲模式只显示当前状态、次数和代理指标。完整报告建议使用离线分析。

## 8. 离线深蹲分析

## 9. 篮球投篮实时模式

```powershell
python main.py `
  --analysis-mode basketball `
  --shot-type set_shot `
  --shooting-side right `
  --camera-view side
```

可选参数：

```text
--shot-type set_shot 或 jump_shot
--shooting-side right 或 left

```

实时篮球模式第一版只显示当前阶段和实时代理指标。完整阶段报告、出手代理、峰值事件、DTW 对齐和一致性分析建议使用离线工具完成。

## 10. 离线篮球投篮分析

先查看可能的投篮候选片段：

```powershell
python -m src.tools.inspect_shot `
  --session outputs\sessions\<session_id> `
  --shooting-side right
```

自动候选只是建议，推荐手动指定投篮片段：

```powershell
python -m src.tools.analyze_shot_session `

  --end-ms 3100 `
  --shot-type set_shot `
  --shooting-side right `
  --camera-view side `
```

```powershell
python -m src.tools.analyze_shot_session `

  --end-ms 3100 `
  --release-ms 1915 `
  --shot-type set_shot `
  --shooting-side right `
  --camera-view side
```

## 13. 多次投篮一致性分析

先分别生成多个篮球投篮报告，然后比较：

```powershell
python -m src.tools.compare_shots `
  --shots outputs\basketball\reports\<shot_a> outputs\basketball\reports\<shot_b> outputs\basketball\reports\<shot_c> `
  --shooting-side right `
  --shot-type set_shot
```

一致性只表示多次动作之间的运动学相似程度，不代表投篮命中率或技术水平。

## 14. 常见工作流

### A. 分析一次深蹲

```text
运行 python main.py
按 C 采集一段深蹲
按 C 停止保存
运行 analyze_squat_session
查看 outputs/squat_reports/<report_id>/report.md
```

### B. 分析一次投篮

```text
运行 python main.py
按 C 采集一次投篮
按 C 停止保存
运行 inspect_shot 查看候选片段
手动选择 start-ms 和 end-ms
运行 analyze_shot_session
查看 outputs/basketball/reports/<report_id>/report.md
```

### C. 与自己的参考动作比较

```text
先录一段你认可的动作
用 create_reference 或 create_shot_reference 建立参考
以后录新的动作
用 compare_session、analyze_squat_session --reference 或 analyze_shot_session --reference 比较
```

## 15. 重要限制

当前系统不能直接测量：

```text
地面反作用力
真实关节力矩
肌肉发力大小
肌电活动
医学风险或运动损伤
篮球真实离手时刻
投篮命中率
动作是否绝对标准
```

`release_proxy_time` 只是基于人体关键点的出手代理时刻。只有未来接入并验证篮球检测后，才可以进一步标记篮球相关的 release candidate。
