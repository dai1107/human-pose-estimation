# 当前任务说明

本项目是 Windows 本地 Python 实时人体姿态检测与运动学分析工具。当前正式保留两个姿态 backend：

1. `mediapipe`：默认主模型，输出 MediaPipe 33 点，适合大多数动作分析、深蹲、投篮、箭步蹲、波比跳远等需要更多身体细节的场景。
2. `yolo-pose`：可选对比/候选模型，输出 COCO 17 点，适合部分快速动作和复杂背景；在有 CUDA 的机器上默认自动使用 GPU。

其他未验证的姿态模型接口不再作为当前项目范围。项目优先保证 MediaPipe 主流程稳定，然后按动作类型在 MediaPipe 与 YOLO Pose 之间选择。

## 核心目标

- 打开摄像头或读取输入视频。
- 实时检测单个人体姿态。
- 显示骨架、关键点、实时 FPS、推理耗时、成功状态和基础反馈。
- 支持录制带标注视频和原始视频。
- 支持同一段视频在不同 backend 下复跑，保存 metrics CSV，做公平对比。
- 支持动作级 backend 自动选择。
- 支持普通实时窗口中按 `B` 在 MediaPipe 与 YOLO Pose 之间热切换，摄像头保持打开。
- 支持短暂 pose 丢失保持与手部遮挡跳点保护，降低实时画面闪断和身体点错跳。

## 当前主入口

实时默认运行：

```powershell
python main.py
```

明确使用 MediaPipe：

```powershell
python main.py --backend mediapipe --smoothing one-euro
```

明确使用 YOLO Pose：

```powershell
python main.py --backend yolo-pose --yolo-pose-model yolo11n-pose.pt --smoothing one-euro
```

动作级自动选择：

```powershell
python main.py --backend auto --action-type rowing --smoothing one-euro
python main.py --backend auto --input-video "HYROX视频\划船机.mp4" --smoothing one-euro
```

实时热切换：

- 在 `--fusion none --person-detector none` 的普通主链路中，窗口打开后按 `B` 可在 `mediapipe` 和 `yolo-pose` 之间切换。
- 切换时只替换姿态 backend，不重新打开摄像头；关键点平滑器会重置。
- 热切换用于实时观察，公平 metrics 对比仍使用固定 backend 分别复跑。

实时稳定性：

- `main.py` 默认 `--pose-hold-frames 5`，短暂识别不到人体时保留上一帧有效骨架，避免画面立即闪成无 pose。
- 默认启用 `--occlusion-guard`，当手腕/手指靠近身体关节且该身体点异常跳变时，短暂保留上一帧稳定位置。
- 如需做最原始模型输出对比，可使用 `--pose-hold-frames 0 --no-occlusion-guard`。

## HYROX 默认策略

基于 `HYROX视频` 中 8 段视频的对比结果，当前策略如下：

| 动作 | 推荐 backend |
|---|---|
| 农夫行走 | `mediapipe` |
| 划船机 | `yolo-pose` |
| 投掷药球 | `mediapipe` |
| 拉雪橇 | `yolo-pose` |
| 推雪橇 | `mediapipe` |
| 波比跳远 | `mediapipe` |
| 滑雪机 | `yolo-pose` |
| 负重箭步蹲 | `mediapipe` |

未知动作默认使用 `mediapipe`。新运动项目必须先录制代表视频，按同一套 metrics 对比后再确定默认 backend。

## GPU 策略

- MediaPipe 继续走 CPU。
- YOLO Pose 默认 `--yolo-device auto`，当前 RTX 4060 环境会解析为 GPU `0`。
- 如需强制 YOLO Pose 走 CPU 对比，可使用：

```powershell
python main.py --backend yolo-pose --yolo-device cpu --smoothing one-euro
```

## 实验性辅助链路

`yolo-roi-mediapipe` 不是正式姿态 backend，也不作为默认推荐方案。它只是一条实验性辅助链路：YOLO 检测人体 bbox，MediaPipe 在 ROI 中继续输出 33 点。

```powershell
python main.py --backend mediapipe --person-detector yolo --fusion yolo-roi-mediapipe --smoothing one-euro
```

当前 HYROX 对比里它没有优于正式默认策略；只有多人、复杂背景或目标锁定场景下才单独测试。它不参与 `--backend auto`，也不支持实时按 `B` 热切换。
