## 1. `src/pose/schema.py`

请实现两个 dataclass：

```python
from dataclasses import dataclass
from typing import Dict, Optional, Tuple


@dataclass
class Keypoint:
    name: str
    x: float
    y: float
    z: Optional[float]
    visibility: float
    confidence: float


@dataclass
class NormalizedPose:
    source: str
    timestamp_ms: int
    image_width: int
    image_height: int
    keypoints: Dict[str, Keypoint]
    bbox: Optional[Tuple[float, float, float, float]]
    overall_confidence: float
```

字段要求：

### `Keypoint`

`name`：
统一关键点名称，例如 `"left_shoulder"`。

`x`：
像素坐标，不是 0 到 1 的归一化坐标。

`y`：
像素坐标，不是 0 到 1 的归一化坐标。

`z`：
如果模型提供 z 值则保留；如果模型不提供，则为 `None`。

`visibility`：
可见性。如果模型没有 visibility，则使用 confidence 代替。

`confidence`：
关键点置信度。如果模型没有单独 confidence，则使用 visibility 或 presence 代替。

### `NormalizedPose`

`source`：
模型来源，只允许使用：

* `"mediapipe"`
* `"yolopose"`

`timestamp_ms`：
当前帧时间戳，单位毫秒。必须保持递增。

`image_width` / `image_height`：
当前图像宽高。

`keypoints`：
字典，key 是统一关键点名称，value 是 `Keypoint`。

`bbox`：
人体框，格式为：

```python
(x1, y1, x2, y2)
```

坐标使用像素坐标。

如果模型没有 bbox，可以为 `None`。

`overall_confidence`：
整个人体姿态的整体置信度。可以用有效关键点 confidence 的平均值计算。

# 五、MediaPipe Adapter 具体要求

MediaPipe 的输出可能来自以下两种形式之一：

1. MediaPipe Tasks API `PoseLandmarkerResult`
2. MediaPipe Solutions API `results.pose_landmarks`

请根据当前项目实际使用的 API 实现。

如果当前项目使用的是 MediaPipe Tasks API，通常结果中会有：

```python
result.pose_landmarks
```

它可能是：

```python
List[List[NormalizedLandmark]]
```

请处理第一个人：

```python
landmarks = result.pose_landmarks[0]
```

对每个映射点执行：

```python
px = landmark.x * image_width
py = landmark.y * image_height
z = landmark.z if hasattr(landmark, "z") else None
visibility = landmark.visibility if hasattr(landmark, "visibility") else 1.0
confidence = visibility
```

注意：

1. MediaPipe 的 x/y 通常是 0 到 1 的归一化坐标。
2. 统一输出时必须转成像素坐标。
3. 如果 landmark 没有 visibility 字段，使用 1.0。
4. 如果关键点坐标缺失或明显非法，可以跳过该点。
5. `overall_confidence` 可以取所有输出关键点 confidence 的平均值。
6. `bbox` 可以根据所有有效关键点的 x/y 最小最大值计算。

bbox 计算方式：

```python
x1 = min(kp.x for kp in keypoints.values())
y1 = min(kp.y for kp in keypoints.values())
x2 = max(kp.x for kp in keypoints.values())
y2 = max(kp.y for kp in keypoints.values())
```

如果没有有效关键点，则返回 `None`。

# 六、YOLOPose Adapter 具体要求

YOLOPose 的输出格式可能因实现不同而不同。请先查看当前项目中 YOLOPose 的真实返回结构，再实现 adapter。

常见情况可能是：

```python
result.keypoints.xy
result.keypoints.conf
result.boxes.xyxy
```

或者：

```python
keypoints: shape = [num_persons, 17, 2]
scores: shape = [num_persons, 17]
boxes: shape = [num_persons, 4]
```

本阶段要求：

1. 支持 COCO 17 点映射。
2. 坐标统一使用像素坐标。
3. 如果 YOLOPose 已经输出像素坐标，不要再乘 image_width / image_height。
4. 如果 YOLOPose 输出归一化坐标，必须转为像素坐标。
5. 每个 keypoint 的 confidence 使用 YOLOPose 的 keypoint score。
6. visibility 可以等于 confidence。
7. `bbox` 优先使用 YOLOPose 的人体框。
8. 如果没有 bbox，则用关键点最小最大值估算。
9. 如果检测到多个人，本阶段先选择 bbox 面积最大的那个人；如果拿不到 bbox，则选择平均关键点置信度最高的人。

多人选择逻辑：

```python
优先级 1：bbox 面积最大的人
优先级 2：平均关键点置信度最高的人
```

请不要在本阶段实现复杂 tracking。
